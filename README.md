# 每日笔记归档器（Daily Note Archiver）

一个 Obsidian 插件，自动管理日记文件夹下的日记文件。每次启动时自动创建今日日记，并清理旧文件至历史记录文件夹。

## 功能

1. **自动创建今日日记** — 每次打开 Obsidian 时，若今日日记不存在则自动创建（支持自定义模板）
2. **自动归档旧文件** — 日记文件夹只保留最近 N 个文件，更早的自动移至 `历史记录/` 子文件夹
3. **完整设置面板** — 路径、保留数、日期格式、日记模板全部可在前台配置

## 安装

### 方法一：手动复制

1. 将整个 `daily-note-archiver` 文件夹放入你 vault 的 `.obsidian/plugins/` 目录
2. 打开 Obsidian → **设置** → **第三方插件** → **关闭安全模式**
3. 在「已安装插件」列表中找到 **每日笔记归档器**，启用即可

### 方法二：通过社区插件市场（将来）

如果在社区市场发布，可直接搜索「每日笔记归档器」安装。

## 配置方式

安装并启用后：

1. 打开 Obsidian 设置
2. 左侧找到「**每日笔记归档器**」
3. 在设置面板中配置你的路径、保留数、日期格式和日记模板

所有配置实时保存，无需手动编辑文件。

## 目录结构示例

```
库根目录/
├── 日记/
│   └── 每日/
│       ├── 2025-03-27.md   ← 最近 N 个保留在此
│       ├── 2025-03-26.md
│       └── 历史记录/        ← 超出 N 的自动移到这里
│           ├── 2025-03-20.md
│           └── ...
└── .obsidian/
    └── plugins/
        └── daily-note-archiver/
            ├── manifest.json
            ├── main.js
            ├── styles.css
            └── README.md
```

## 设置项说明

| 设置项 | 类型 | 说明 |
|--------|------|------|
| 日记文件夹路径 | 文本框 | 存放日记的目录，相对于库根目录 |
| 历史记录文件夹路径 | 文本框 | 旧日记移入的目录 |
| 保留文件数 | 滑块（1-60） | 当前文件夹保留多少个文件 |
| 日期格式 | 文本框 | moment.js 格式（`YYYY-MM-DD` / `YYYYMMDD` / `YYYY年M月D日`） |
| 标题行 | 文本框 | 文件首行，支持 `{{date}}` 占位符 |
| 正文模板 | 多行文本框 | 标题之后的内容，支持 `{{date}}` 占位符 |
| 自动创建 / 自动归档 | 开关 | 是否在启动时自动执行 |

## 手动触发

打开设置面板点击「**立即执行**」按钮，或在控制台执行：

```js
app.plugins.plugins["daily-note-archiver"].runNow()
```

## 从旧版迁移

如果你之前直接修改过 `main.js` 中的默认值，安装新版后请通过设置面板重新配置——设置面板的修改会存入 `data.json`，优先级高于代码中的默认值。
